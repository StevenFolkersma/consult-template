# consult-template

A lightweight alternative to [yasnippet](https://github.com/joaotavora/yasnippet) for simple, static text templates in Emacs.

Unlike yasnippet, there is no snippet syntax, no tab stops, no expansion logic. Templates are plain strings. The focus is on quickly storing and retrieving boilerplate text — email sign-offs, code skeletons, org headings, anything you type repeatedly.

Selection is powered by [consult](https://github.com/minad/consult), with live preview of the template body as you narrow and an annotated truncated preview in the margin.

---

## Features

- Insert templates via a consult picker with live preview
- Define new templates interactively from a region or typed input
- Templates persist across sessions in a plain Elisp file
- Works with [embark](https://github.com/oantolin/embark) — capture any region as a template without leaving your buffer

---

## Installation

### Clone and add to load path

```bash
git clone https://github.com/StevenFolkersma/consult-template.git ~/.config/emacs/consult-template
```

```elisp
(add-to-list 'load-path "~/.config/emacs/consult-template")
(require 'consult-template)
```

### `use-package` with `:vc`

Requires Emacs 29.1+ or the `vc-use-package` package on earlier versions.

```elisp
(use-package consult-template
  :vc (:url "https://github.com/StevenFolkersma/consult-template.git" :rev :newest)
  :demand t)
```

## Configuration

All settings are optional. The only one you are likely to want is a custom path for the templates file.

```elisp
(use-package consult-template
  :vc (:url "https://github.com/StevenFolkersma/consult-template.git" :rev :newest)
  :demand t
  :custom
  ;; Where templates are persisted. Defaults to templates.el in
  ;; user-emacs-directory. Set this before the package loads.
  (consult-templates-file "~/.config/emacs/my-templates.el")
  :bind
  ("C-c t i" . consult-template-insert)
  ("C-c t d" . consult-template-define))
```

### Embark integration

Bind `consult-template-define` to `embark-region-map` to capture any region as a template with a single keypress:

```elisp
(with-eval-after-load 'embark
  (keymap-set embark-region-map "T" #'consult-template-define))
```

When called via `embark-act` on a region, the region text is pre-filled into the body prompt (you can edit it before confirming with RET). The name prompt is always left for you to type.

## Usage

### Insert a template — `M-x consult-template-insert`

Opens a consult picker listing all defined templates. Narrow by name, see a truncated preview in the annotation, and get a live overlay preview of the full body in the buffer as you move through candidates. Confirm with RET to insert at point.

### Define a template — `M-x consult-template-define`

With an **active region**: the region text is pre-filled as the template body. You can edit it in the prompt before confirming, then provide a name.

Without a region: you are prompted for the body text, then the name.

If a template with the same name already exists, you are asked to confirm before overwriting.

Templates are saved immediately to `consult-templates-file` and are available in future sessions without any extra steps.

---

## Templates file

Templates are stored as a plain Elisp `setq` form in `consult-templates-file` (default: `templates.el` in `user-emacs-directory`). The file is written automatically on every save and loaded automatically when the package loads.

Example file:

```elisp
;; Auto-generated by consult-templates.el -- do not edit by hand
(setq consult-templates
'(("sign-off" . "Best regards,\nJohn Doe")
  ("org-header" . "#+title: \n#+date: \n#+author: John Doe\n")
  ("todo" . "TODO: ")))
```

The file is human-readable and can be edited by hand if needed, though it will be overwritten on the next `consult-template-define` call.
